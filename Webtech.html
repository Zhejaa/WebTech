HYPERTEXT TRANSFER PROTOCOL

			
				HTTP is short for HyperText Transfer Protocol. HTTP is the underlying protocol used by World Wide Web and this protocol defines how messages
			are formatted and transmitter, and what actions Web servers and browsers should take in response to various commands.
				
				Reference: http://www.webopedia.com/TERM/H/HTTP.html
					 Read More 
				
				Application layer used primarily to retrieve hypertext documents and resources on the World Wide Web.
				
				Jointly developed by the W3C and the IETF.
				
				Protocol is a set of rules need to be followed.
				
				HTTP typically runs on top of TCP/IP, using TCP port 80 by default.
				
				HTTP resources are identified using URLs, specifically, HTTP URLs.
				
				
				HTTP is based on client-server architecture:
					Clients
						Web browsers, web crawlers, email clients, other end user tools and applications.
					
					Servers
						Origin servers, proxy servers, gateways, tunnels.
						
				HTTP provides support for other functionalities such as cache control, content media type (MIME) specification, language and character set
				specification, content/transfer coding, client-server protocol negotiations, etc.
