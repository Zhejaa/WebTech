CSS Versions

  • HTTP typically runs on top of TCP/IP, using TCP port 80 by default (TCP port 443 for HTTPS).
  • HTTP resources are identified using URIs (specifically HTTP URLs).
  • HTTP is based on client-server architecture.
    • Clients (AKA User Agents (UA)).
      • Web browsers
      • Web crawlers
      • Email clients
      • Other end user tools and applications
    • Servers
      • Origin servers
      • Proxy servers
      • Gateways
      • Tunnels
  • HTTP uses a request-response standard protocol.
    • Client sends an HTTP request to the server.
    • Server processes the request and replies with an HTTP response message.
  • HTTP is a stateless communications protocol.
    • Servers DO NOT keep information about clients between requests.
    • Web applications affect session tracking using mechanisms such as cookies (a very small text file) on URL-encoded session information to keep track of related client requests.
  • HTTP provides support for other functionalities such as cache-control (cache: Local storage/copy of a resource that is fetched from a server), content media type (MIME) specification, language and character set specification, content/transfer coding, client-server protocol negotiations, persistent connections, request pipelining etc.

CSS Statements

  • Standard Methods
    • GET – retrieved resource is returned in the body of response as entity
    • HEAD – same as GET, except the message body is not included in the response
    • POST – submit data to server, usually a script
    • PUT – send data a specified URL to a server
    • DELETE – specify resource and delete from server
    • OPTIONS - request for information about the communication options
    • TRACE – requests loop-back of the request message
    • CONNECT – request the establishment of a tunnel
  • Idempotent Methods
    • GET, HEAD, PUT, DELETE
  • Extension Methods
    • PROPFIND
    • PROPPATCH
    • MKCOL
    • COPY
    • MOVE
    • LOCK
    • UNLOCK

CSS Rule Precedence

  • Informational (1xx)
    • 100 Continue -used when asking if you can continue
    • 101 Switching Protocols -used when the client requested for an update
  • Success (2xx)
    • 200 Ok
    • 201 Created
    • 202 Accepted
    • 203 Non-Authorative Information
    • 204 No Content -if the delete is successful
    • 205 Reset Content -used to tell the client to reset the data entry form
    • 206 Partial Content
  • Redirection (3xx)
    • 300 Multiple Choices
    • 301 Moved Permanently -to tell the client that resource has been moved
    • 302 Found -redirects from a server
    • 303 See other
    • 304 Not Modified
    • 305 Use proxy -you cannot access from the server
    • 307 Temporary Redirect
  • Client Error (4xx)
    • 400 Bad request
    • 401 Unauthorized
    • 402 Payment Required -providing payment by accessing resources
    • 403 Forbidden -You’re not allowed to access

    • 409 Conflict
